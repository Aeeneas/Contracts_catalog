## План реализации портала-каталогизатора (для Gemini CLI)

Этот план детализирует следующие шаги, которые я, как Gemini CLI, предприму для дальнейшей реализации проекта, основываясь на утвержденном Техническом задании.

---

**Этап 1: Анализ существующей структуры и подготовка данных (ЗАВЕРШЕН)**
*   Изучение структуры файлов: Выполнено, выявлены паттерны и сложности.
*   Сбор примеров документов: Выполнено, проведен анализ примерного документа.

**Этап 2: Проектирование базы данных и API (ЗАВЕРШЕН)**

**2.1. Разработка схемы БД:**
*   Создан SQL-скрипт для создания таблицы `contracts` (PostgreSQL). **ЗАВЕРШЕН**
*   Создана модель SQLAlchemy для `Contract`. **ЗАВЕРШЕН**

**2.2. Разработка Backend API (Python + FastAPI):**
*   Инициализация проекта FastAPI (создание директории `backend`, виртуального окружения, установка `fastapi`, `uvicorn`, `python-multipart`, `sqlalchemy`, `psycopg2-binary`, `python-dotenv`, `python-docx`, `pymupdf`). **ЗАВЕРШЕН**
*   Создание `main.py` с заглушками для основных эндпоинтов. **ЗАВЕРШЕН**
*   Создание `config.py` для загрузки переменных окружения. **ЗАВЕРШЕН**
*   Создание `database.py` для настройки подключения к БД и определения ORM-модели. **ЗАВЕРШЕН**
*   Создание `create_db.py` для инициализации таблиц БД. **ЗАВЕРШЕН**
*   Создание `text_extractor.py` для извлечения текста из документов. **ЗАВЕРШЕН**
*   Создание `ai_service.py` для интеграции с Gemini API. **ЗАВЕРШЕН**
*   Создание `contract_utils.py` для генерации уникальных номеров. **ЗАВЕРШЕН**
*   **Ожидаемые действия пользователя:** Настроить PostgreSQL, создать базу данных, обновить `DATABASE_URL` и `GEMINI_API_KEY` в `.env`, запустить `create_db.py` для создания таблиц. **ВЫПОЛНЕНО ПОЛЬЗОВАТЕЛЕМ**
*   **Интеграция БД в `main.py`:** Модификация `main.py` для использования `get_db` в эндпоинтах и реализация базовых CRUD-операций для контрактов. **ЗАВЕРШЕН**
*   **Реализация полной логики эндпоинта `/upload`:**
    *   Обработка и временное хранение файлов. **ЗАВЕРШЕН**
    *   Извлечение текстового слоя из документов. **ЗАВЕРШЕН**
    *   Интеграция с AI (Gemini API). **ЗАВЕРШЕН**
    *   Генерация `Уникального_номера_договора`. **ЗАВЕРШЕН**
    *   Проверка на дубликаты. **ЗАВЕРШЕН**
    *   Формирование файловой структуры и перемещение файлов. **ЗАВЕРШЕН**
    *   Сохранение данных в БД. **ЗАВЕРШЕН**
    *   Обработка ошибок и логирование (частично встроено). **ЗАВЕРШЕН**

---

**Этап 3: Разработка веб-интерфейса (Frontend) (ЗАВЕРШЕН)**

*   **Настройка проекта:** Инициализация нового React-проекта в директории `frontend`. **ЗАВЕРШЕН**
*   **Компонент загрузки:** Разработка компонента Drag&Drop для загрузки файлов, взаимодействующего с `/upload` эндпоинтом. **ЗАВЕРШЕН**
*   **Компонент таблицы/поиска:** Разработка UI для отображения списка договоров, фильтрации и поиска, с взаимодействием с бэкенд `/contracts` эндпоинтом. **ЗАВЕРШЕН**
*   **Компонент деталей договора:** Разработка UI для просмотра детальной информации о выбранном договоре. **ЗАВЕРШЕН**
*   **Интеграция с API:** Подключение фронтенда к разработанным бэкенд API. **ЗАВЕРШЕН**

**Этап 4: Тестирование и развертывание (ПЛАНИРУЕТСЯ)**
*   **Unit/Integration тесты:** Написание тестов для бэкенда и фронтенда.
*   **Функциональное тестирование:** Проверка всех пользовательских сценариев.
*   **Развертывание:** Подготовка инструкций по развертыванию приложения.
